代码格式规范
1.Microsoft提供的命名空间放在一起,其它的命名空间则空一行写在一起,与Microsoft的区分开来.
2.变量使用时要先定义,变量定义时的定义代码要写在一起.(即是:定义块写在一起,逻辑块写在一起)

----------------------------------------------------------------------------------------------
类、接口、枚举命名规范

类命名规范
[C]类名(注：第一个字母大写)
如:public CDataBase{}

接口命名规范
[I]接口名(注：第一个字母大写)
如:public IDBManager{}

枚举命名规范(注：第一个字母大写)
[E]枚举名称
如:public EDriveIndex{}

----------------------------------------------------------------------------------------------
控件类、字段静态类命名规范

控件类
[Ctl]类名
如：public CtlMenu{}

字段静态类
[F]类名(注：第一个字母大写)
如：public FUser{}

----------------------------------------------------------------------------------------------
函数命名规范

方法命名规范
方法名称(注：第一个字母大写)
如：public string GetAppValue(){}

----------------------------------------------------------------------------------------------
变量命名规范

成员变量命名规范(方法外面的变量)
[m_] [类型] 变量名称(注：第一个字母小写)
如：m_strName

变量命名规范(方法里面的变量)
[类型] 变量名称(注：第一个字母小写)
(备注:函数返回值变量一律用result表示)
如：strName

控件命名规范
[m_][类型] 控件名称(注：第一个字母小写)
如：m_txtPassworld

----------------------------------------------------------------------------------------------
参数命名规范

参数命名规范
[p_][类型] 参数名称(注：第一个字母小写)
如：public string GetAppValue(String p_strKey){}

----------------------------------------------------------------------------------------------

注：
方法返回结果无论是什么类型一律用 result 表示

Entity ent

Field fid

Enum enu
