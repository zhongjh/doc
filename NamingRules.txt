----------------------------------------------------------------------------------------------
C#命名规则
----------------------------------------------------------------------------------------------
代码格式规范
1.Microsoft提供的命名空间放在一起,其它的命名空间则空一行写在一起,与Microsoft的区分开来.
2.变量使用时要先定义,变量定义时的定义代码要写在一起.(即是:定义块写在一起,逻辑块写在一起)

----------------------------------------------------------------------------------------------
变量命名规范
[m_] [类型] 变量名称(注：第一个字母小写)
(备注:函数返回值变量一律用result表示)
如：m_strName

控件命名规范
[m_] [类型] 控件名称(注：第一个字母小写)
如：m_txtPassworld

方法命名规范
方法名称(注：第一个字母大写)
如：public string GetAppValue(){}

参数命名规范
[p_] [类型] 参数名称(注：第一个字母小写)
如：public string GetAppValue(p_strKey){}

类命名规范
类名(注：第一个字母大写)
如:public DataBase{}

接口命名规范
[I]接口名(注：第一个字母大写)
如:public IDBManager{}

枚举命名规范(注：第一个字母大写)
[E]枚举名称
如:public EDriveIndex{}

数据库命名规则
------------------------------------------------------------------------------------------
每个表都要有一个自动ID(它是与业务无关)且为主键

主键的值不能被修改

命名规范
表名
[t_][用途_]表名(r关系表,s基本表,b业务表,w网站表)
如：t_s_Users

字段
[m_]字段名
如：m_UserName

------------------------------------------------------------------------------------------

XML文件设计规范

节点
节点名称(第一个字母小写)

节点值
节点值(第一个字母大写)

节点属性
节点属性名称(第一个字母小写)

节点属性值
节点属性值(第一个字母大写)
